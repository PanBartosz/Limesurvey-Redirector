Polski

Krótka instrukcja dla użytkownika tras.

Szybki workflow

  1. Zaloguj się i otwórz Routes.
  2. Wpisz Name i Slug.
  3. Wybierz Instance.
  4. Dodaj ankiety w sekcji Surveys in this route, po jednej ankiecie w wierszu.
  5. Wybierz Algorithm.
  6. Jeżeli używasz algorytmu ważonego, ustaw Weight dla każdej ankiety.
  7. W razie potrzeby ustaw Fallback URL, przekazywanie parametrów, pending buffer lub stickiness.
  8. Kliknij Create Route.
  9. Skopiuj Public URL z karty trasy i udostępniaj tylko ten link.

Jaki algorytm wybrać

  • Completed Fuzzy: najlepszy domyślny wybór dla kopii tej samej ankiety.
  • Least Completed: najprostsze wyrównywanie po liczbie ukończonych odpowiedzi.
  • Weighted Completed / Weighted Fuzzy: używaj tylko wtedy, gdy część ankiet ma dostać więcej ruchu.

Na co uważać

  • Wszystkie ankiety w trasie powinny być aktywne.
  • Łącz tylko ankiety, które są faktycznie kopiami tego samego badania.
  • Slug musi być unikalny.
  • Po zapisaniu zawsze otwórz Public URL testowo.

English

Short route-user guide.

Quick workflow

  1. Log in and open Routes.
  2. Enter Name and Slug.
  3. Select the correct Instance.
  4. Add surveys in Surveys in this route, one survey per row.
  5. Select the Algorithm.
  6. If you use a weighted algorithm, set Weight for each survey.
  7. If needed, configure Fallback URL, query forwarding, pending buffer, or stickiness.
  8. Click Create Route.
  9. Copy the Public URL from the route card and share only that link.

Which algorithm to start with

  • Completed Fuzzy: best default for several clones of the same survey.
  • Least Completed: simplest balancing by completed response count.
  • Weighted Completed / Weighted Fuzzy: use only when some surveys should intentionally receive more traffic.

What to double-check

  • All surveys in the route should be active.
  • Only group surveys that are actual clones of the same study.
  • Slug must be unique.
  • Always smoke-test the Public URL after saving.
